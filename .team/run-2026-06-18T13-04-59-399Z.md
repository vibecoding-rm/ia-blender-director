# Análisis: IA Blender Director

## 1. Qué es y su objetivo

Pipeline **local** que convierte una idea en lenguaje natural en un Short animado vertical (estilo claymation), pensado para una serie de sátira de noticias: *"El Noticiero de La Cotorra"*. La premisa de diseño (README.md:16, docs/architecture.md:1-8) es que **la IA dirige pero no ejecuta código arbitrario**: produce un `ShotSpec` JSON validado, y Blender es la "fuente de verdad" que renderiza. Objetivo de negocio: publicar Shorts 9:16 sin depender de generadores de video cerrados de pago.

El flujo completo: `prompt → planner (LLM/fallback) → ShotSpec validado → Blender render + control passes → Vision Critic → ComfyUI (estilización, opcional) → postproducción (gancho, TTS, subtítulos, SFX, transiciones xfade/Ken Burns) → Short final`.

## 2. Arquitectura y componentes

**`src/ai_blender_director/`** — paquete Python (núcleo):
- `models.py` — `ShotSpec` Pydantic congelado (`frozen=True`), con `TransitionSpec` y `Resolution/CameraSpec/AssetRefs`. Contrato central, con `from_dict()` retro-compatible.
- `planner.py` — el "Director Agent": `plan_scene()` llama a OpenRouter (modelo `google/gemini-2.0-flash-001`) y, si no hay API key o falla, cae a `_fallback_plan()`, un generador heurístico por palabras clave bastante elaborado (planos establishing→action→close_up, lógica especial para el noticiero).
- `generator.py` — wrapper fino de 1 plano sobre `planner.plan_scene` (antes duplicaba lógica; ya refactorizado).
- `postproduction.py` — ensamblaje con **`ffmpeg-python`**: `xfade` entre clips, `zoompan` (Ken Burns) en planos `static`, narración TTS, subtítulos `.ass` quemados, mezcla de audio. Es el módulo central de la tarea reciente (ORIGINAL_REQUEST.md).
- `critic.py` — `VisionCritic`: intenta crítica multimodal vía LLM y cae a 6 reglas heurísticas sobre beauty+mask (distancia, encuadre, luz, sobreexposición, contraste, recorte).
- `server.py` — FastAPI: endpoints `/api/director/plan`, `/api/director/render`, `/api/pipeline`, WebSocket de logs, caché en memoria + SQLite.
- `db.py` / `index.py` / `jobs.py` — persistencia SQLite (`renders/jobs.db`) de jobs y planes; `index.py` mantiene firmas con `index_path` pero ya usa SQLite por dentro.
- `cli.py` + `commands/*` — CLI `ai-blender-director` con subcomandos (generate, render, critic, batch, serve…).
- `tts.py`, `sfx.py`, `subtitles.py`, `branding.py`, `comfy/client.py` — utilidades de postproducción y cliente ComfyUI.

**`scripts/blender/ai_director/`** — código que corre *dentro* de Blender (camera, lighting, materials, animation, passes, render…), invocado como subproceso `blender --background --python render_shot.py`.

**`web/`** — UI estática (index.html + app.js) servida por FastAPI. **`workflows/comfy/`** — workflow ComfyUI (futuro). **`assets/`** — personajes/escenarios con `asset.json` por versión (los `.glb` son placeholders). **`tests/`** — 16 archivos, **86 tests, todos en verde** (los corrí). **`.agents/`** — andamiaje de una orquestación multi-agente previa (briefings, handoffs, crons).

## 3. Estado y madurez

**Maduro y sólido:** el contrato `ShotSpec`, la validación, el sistema de jobs reproducibles, la postproducción ffmpeg-python (transiciones + Ken Burns implementadas y testeadas: tests/test_transitions.py verifica duración con `ffprobe`), y la suite de tests (86/86). def plan_scene(

def plan_shots(
    prompt: str,
    n_shots: int = 3,
    *,
    duration_seconds: int = 4,
    fps: int = 24,
    resolution: dict[str, int] | None = None,

    return scene_data["shots"]


def write_shot_plan(
    prompt: str,
    output_dir: Path,
    n_shots: int = 3,
    *,
    duration_seconds: int = 4,
    fps: int = 24,
    resolution: dict[str, int] | None = None,
    precomputed_shots: list[dict] | None = None,
) -> list[Path]:
    """Genera y guarda todos los planos del plan. Devuelve las rutas guardadas."""
    output_dir.mkdir(parents=True, exist_ok=True)
    if precomputed_shots:
        shots = precomputed_shots
    else:
        shots = plan_shots(prompt, n_shots, duration_seconds=duration_seconds, fps=fps, resolution=resolution)
    
    base_slug = _slug(prompt)
    paths: list[Path] = []
    for i, shot in enumerate(shots, start=1):
        role = shot.get("_shot_role", f"shot_{i:02d}")
        filename = f"{base_slug}_{role}_{i:02d}.json"
        shot_data = {k: v for k, v in shot.items() if not k.startswith("_")}
        path = output_dir / filename
        with path.open("w", encoding="utf-8") as f:
            json.dump(shot_data, f, indent=2, ensure_ascii=False)
            f.write("\n")
        paths.append(path)
    return paths

# Análisis: IA Blender Director

## 1. Qué es y su objetivo

Pipeline **local** que convierte una idea en lenguaje natural en un Short vertical animado (estilo claymation), para una serie de sátira de noticias: *"El Noticiero de La Cotorra"*. La premisa de diseño (`README.md`, `docs/architecture.md`) es que **la IA dirige pero no ejecuta código arbitrario**: produce un `ShotSpec` JSON validado y Blender es la "fuente de verdad" que renderiza. Objetivo de negocio: publicar Shorts 9:16 sin depender de generadores de video cerrados de pago.

**Flujo:** `prompt → planner (LLM/fallback) → ShotSpec validado → Blender render + control passes → Vision Critic → ComfyUI (estilización, opcional) → postproducción (gancho, TTS, subtítulos, SFX, transiciones xfade/Ken Burns) → Short final`.

## 2. Arquitectura y componentes

**`src/ai_blender_director/`** — núcleo Python:

**🟠 P4 — Resolución de Blender incoherente entre CLI y servidor.**

Resultado: funciona en la máquina del autor pero falla en instalaciones limpias o Windows con ruta custom — sin un preflight claro. **Fix:** unificar en `settings.blender_executable` con fallback a PATH, y añadir un preflight check (Blender/FFmpeg/ComfyUI/modelos) que falle temprano y con mensaje útil.

**🟡 P5 — Riesgo de contrato de assets + higiene de repo.**
